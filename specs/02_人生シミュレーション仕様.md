## 0. 用語の前提（共通ベースライン準拠）

本書で用いる主な用語は共通ベースラインの用語集に従う。とくに本書で頻出するもの：

- **一族**＝継承する家系全体／**当主**＝現操作キャラ／**家名**＝一族の名前（プレイヤー命名）
- **代替わり**＝大往生による次代継承（家系継続）／**大往生**＝寿命到来の自然死（失敗でなく完結）／**滅亡**＝週間目標未達による家系断絶（01で定義）
- **人生ステージ**＝誕生→幼少期→学生期→青年期→壮年期→老年期→大往生
- **ステータス**＝健康／活力／人望／財産（各0〜100の整数）
- **歩数蓄積量**＝当代の累計歩数（当主が誕生してから現在までに歩いた歩数の総和）
- **遺伝**＝外見・性格の一部継承／**家宝**＝よく歩いた世代が次代に遺す恩恵

確定事項（共通ベースラインより本書に関係する範囲）：

- 指標は**歩数のみ**（距離・カロリーは使わない）
- 週の締めは**日曜24時固定**、目標変更は**翌週反映・当週不可**（01）
- ステージ進行・寿命は**現実時間ベース**で進む（歩いていなくても時間は進む）
- 救済モードなし（病気・怪我・出張でも救済しない。自衛手段は「翌週からの目標引き下げ」のみ＝01）
- 技術前提：**Android／Kotlin／Jetpack Compose／Health Connect／Room**。初期は**無料**で提供。

## 1. 確定事項（要確認事項への回答）

旧版（v1.0）で未確定として列挙していた各論点は、ユーザー決定により以下のとおり確定した。本書は本節の確定値を全面採用し、本文中の「暫定」表記は廃止する。

### 1-1. 一代の長さ（要確認#4・共通ベースラインB-1）
- **確定：標準寿命 = 20日**。固定ステージ（幼少期〜壮年期）合計13日＋老年期で構成。短回転で世代交代の達成感を優先（看取りと回転のバランス）。

### 1-2. 寿命変動幅（共通ベースラインB-1）
- **確定：±6日で14〜26日に変動**。健康度・歩数達成率で老年期長を伸縮させて吸収（§4）。

### 1-3. 歩数→ステータスの変換係数（基準＝目標相対）
- **確定：固定基準歩数（旧8,000歩）は廃止**。中立点はその週の週間目標（activeGoal）の日割り `g = activeGoal ÷ 7` とする（目標相対）。`d = (steps_today − g) / g` を `clamp(−1.0, +2.0)` し、活力/健康/人望/財産へ反映（§5-2）。目標が下がれば達成しやすく、上がれば難しくなる自己調整型。

### 1-4. ステータス初期値（誕生時）
- **確定：健康60／活力50／人望50／財産40**。健康を高め・財産を低めにして「歩いて積み上げる」余地を残す。

### 1-5. 遺伝の継承率
- **確定：ステータス継承20%（基礎超過分のみ・健康/人望/財産）／外見・性格形質は各50%継承**。活力は継承せず初期値50（短期指標のため）。

### 1-6. 家宝の効果量（目標相対化）
- **確定：絶対歩数閾値（旧100k/250k/400k）は廃止**。当代の生涯通算達成率（生涯累計実歩数÷生涯期待累計歩数）でランク判定：金（≥110%）+10／銀（80〜110%）+6／銅（<80%）+3。低でも必ず銅以上＝必ず何か遺す。1代限り・非累積（§7-4）。

### 1-7. 性別・名前の生成方式（要確認#5・共通ベースラインB-2）
- **確定：家名のみプレイヤー入力、当主名・性別は自動生成（任意で改名可）**。家名のみ命名負荷を負わせ、代替わりごとの命名疲れを防ぐ。
  - 補足：03/04の**イベント性差トグル**が「あり」のとき、自動生成された性別がイベント分岐に使われる。「なし」のときは性別は外見上のみで分岐に影響しない（メカニクス本体は03で定義）。

### 1-8. 寿命到来と週判定の優先順位（参照のみ）
- 本書では扱わず**01側で定義**する。本書は「寿命が来たら大往生し代替わりする」というメカニクスのみ規定し、同一タイミングで週判定（滅亡）と大往生が衝突した場合の優先順位は01に委ねる（§4-5に接続メモ）。

## 2. 時間軸の基本設計

### 2-1. 進行の原則
- 人生ステージと寿命は**現実時間ベース**で進行する。**歩いていなくても時間は進む**（時間を止める/巻き戻す手段はない）。
- 歩数は「時間の進み」ではなく「人生の質（健康・寿命・ステータス・イベント結果）」に作用する。
- 時間の最小単位は**1日**（日次バッチで日が進む）。日付の境界・週の境界判定の正は**01**に従う。

### 2-2. 人生の長さ（標準）
- 標準寿命 = **現実20日**（§1-1）。健康度・歩数達成率により最終的に**14〜26日**に変動（§1-2、§4）。
- 「1日」は現実の1日に対応する。アプリ未起動でも日次バッチ（端末ローカル）で経過日数を補完する。

## 3. 人生ステージ設計

### 3-1. ステージ別標準日数

標準寿命20日を前提とした、各ステージの**標準日数（現実日数）**。誕生は0日（誕生＝その瞬間のイベント、幼少期初日に内包）。

| ステージ | 標準日数 | 累積日（開始〜終了） | 意味づけ |
|---|---:|---|---|
| 誕生 | 0日 | 0日目（瞬間） | 当主の生成・遺伝/家宝の適用。イベント点。 |
| 幼少期 | 3日 | 1〜3日目 | **基礎**。健康・活力の土台づくり。歩く習慣の助走。 |
| 学生期 | 3日 | 4〜6日目 | **進路**。学業/進学の方向づけ。人望の伸び始め。 |
| 青年期 | 3日 | 7〜9日目 | **就職・結婚**。財産と人望が動く転機。 |
| 壮年期 | 4日 | 10〜13日目 | **家庭・事業の充実**。財産が最も動く安定成長期。 |
| 老年期 | 7日（標準） | 14〜20日目 | **健康寿命部分**。寿命調整の主舞台（§4）。歩数蓄積の集大成。 |
| 大往生 | ―（瞬間） | 寿命到来時 | 一生の完結。代替わりへ。 |

合計：0+3+3+3+4+7 = **20日**（標準）。固定ステージ（幼少期〜壮年期）合計＝**13日**。

> 注：03（人生イベント仕様）のイベント発生条件は**このステージ区分・累積日**を前提とする。ステージ日数の変更は03のイベントタイミングに直結するため、両書で同一値を維持すること。

### 3-2. ステージ進行のルール
- ステージは累積経過日数が各ステージの終了日に達した時点で次へ遷移する。
- 寿命変動（§4）は**老年期の長さで吸収**する。すなわち幼少期〜壮年期の標準日数（合計**13日**）は原則固定し、老年期の日数を伸縮させて最終寿命を14〜26日に収める。
  - **老年期日数 = max(1, 最終寿命 − 13)**（範囲1〜13日）。
  - 例：最終寿命26日（上限）→ 老年期 = 26 − 13 = **13日**。
  - 例：最終寿命20日（標準）→ 老年期 = 20 − 13 = **7日**。
  - 例：最終寿命14日（下限）→ 老年期 = 14 − 13 = **1日**（下限。これ未満には縮めない。健康が極端に低くても壮年期までは保証）。
- **老年期突入＝経過13日完了時（＝14日目開始時）**。この時点で最終寿命を確定する（§4-2）。
- ステージ進行は時間ベースのため、歩数で前倒し/後ろ倒しはしない。歩数が効くのは「最終寿命の確定（＝老年期長）」と「各ステージ中のステータス・イベント結果」である。

## 4. 寿命の設計

### 4-1. 設計方針
- **死は失敗ではなく一生の完結**。寿命到来＝大往生＝家系は継続（代替わり）。失敗は滅亡（01）のみ。
- **よく歩いた当主＝健康寿命が延びる**。歩数達成率と健康度が高いほど最終寿命が長くなる。
- 寿命は標準20日を基準に、**健康度（健康ステータス）**と**歩数達成率（目標相対）**で14〜26日に変動。

### 4-2. 最終寿命の確定タイミング
- 各ステージの標準日数は固定だが、**最終寿命は老年期突入時（経過13日完了時＝14日目開始時）に確定**させる。
- 理由：幼少期〜壮年期を通じて積んだ健康・歩数の成果を、老年期長＝健康寿命として反映するため。老年期入りまでにどれだけ歩いたかが寿命に効く、という直感に合う。

### 4-3. 最終寿命の計算モデル（目標相対）

老年期突入時点（経過13日）の**健康ステータス H（0〜100）**と**累計実歩数 S（歩）**から、最終寿命 `lifespan_days` を算出する。歩数評価は固定基準ではなく**その週の週間目標を日割りした g** に対する達成率で行う（目標相対）。

```text
# 入力
H        : 老年期突入時の健康ステータス（0〜100）
S        : 老年期突入時の累計実歩数（歩）＝歩数蓄積量

# 基準（確定値）
BASE        = 20   # 標準寿命（日）
STAGE_FIXED = 13   # 幼少期〜壮年期の固定合計（日）
MIN_LIFE    = 14   # 寿命下限（日）
MAX_LIFE    = 26   # 寿命上限（日）
ELAPSED     = 13   # 老年期突入までの経過日数（固定）

# 日次目標歩数 g：その週の週間目標(activeGoal)を7で割った値。
#   g = activeGoal ÷ 7
# 期待累計 expected_total：各経過日の g を合算した値（目標相対）。
#   週間目標が一定であれば expected_total = g × 13。
#   週ごとに目標が変わる場合は各経過日のgの総和を用いる（本書が正＝§下記注）。

# 1) 健康ボーナス：健康50を中立として ±4日の範囲で線形
#    H=100 → +4日 / H=50 → 0日 / H=0 → -4日
health_bonus = (H - 50) / 50.0 * 4.0      # 単位：日（-4〜+4）

# 2) 歩数ボーナス：期待累計に対する達成率で ±3日
#    rate = S / expected_total
#    rate>=1.5 で +3日上限、rate<=0.5 で -3日下限（線形クランプ）
rate = S / expected_total
steps_bonus = clamp((rate - 1.0) / 0.5 * 3.0, -3.0, +3.0)  # 単位：日（-3〜+3）

# 3) 合算してクランプ
raw = BASE + health_bonus + steps_bonus
lifespan_days = round(clamp(raw, MIN_LIFE, MAX_LIFE))   # 14〜26

# 4) 老年期の長さに反映（下限1日）
oldage_days = max(1, lifespan_days - STAGE_FIXED)        # 1〜13
final_lifespan = STAGE_FIXED + oldage_days              # 実効寿命（14〜26）
```

`clamp(x, lo, hi)` は `min(max(x, lo), hi)`。

> **期待累計歩数の式は本書（02）が正**：`期待累計 = g × 経過実日数`（g＝週間目標÷7）。03はこの式を参照すること。

### 4-4. 計算例（確定値での確認）

以下は週間目標が一定で g = 8,000歩/日（週間目標56,000歩）として期待累計 = 8,000×13 = **104,000歩** を基準にした例。週間目標が異なれば g・期待累計はそれに応じて変わる（目標相対）。

| ケース | H（健康） | S（累計実歩数） | health_bonus | rate | steps_bonus | lifespan_days | 老年期日数 |
|---|---:|---:|---:|---:|---:|---:|---:|
| よく歩いた | 90 | 200,000 | +3.2 | 1.92 | +3.0（上限） | 26（上限） | 13日 |
| やや好調 | 70 | 130,000 | +1.6 | 1.25 | +1.5 | 23 | 10日 |
| 標準 | 60 | 104,000 | +0.8 | 1.00 | 0.0 | 21 | 8日 |
| 平均的 | 50 | 90,000 | 0.0 | 0.87 | -0.8 | 19 | 6日 |
| あまり歩かず | 35 | 60,000 | -1.2 | 0.58 | -2.5 | 16 | 3日 |
| ほぼ歩かず | 15 | 25,000 | -2.8 | 0.24 | -3.0（下限） | 14（下限） | 1日 |

> 「よく歩いた当主」は健康・歩数の両ボーナスが効いて上限26日まで延び、老年期がもっとも長くなる（看取りまでの猶予が増える）。「ほぼ歩かず」でも下限14日は保証され、老年期1日として必ず代を完結できる（いきなり代が途切れる理不尽は避ける。滅亡＝01とは別概念）。

### 4-5. 寿命到来時の処理と接続メモ
- 最終寿命の最終日（`final_lifespan` 日目、14〜26日目）の日次バッチ完了時に**大往生**を発生させ、§7の代替わり処理に入る。
- 大往生と週間目標判定（滅亡＝01）が同一タイミング（例：日曜24時＝週締めと寿命到来が重なる）で衝突しうる。**その優先順位は01側で定義**する。本書は「大往生は家系継続、滅亡は家系断絶」というメカニクスの差のみ規定する。

## 5. ステータス数値設計

### 5-1. ステータスと初期値

各ステータスは0〜100の整数。誕生時の**基礎初期値（確定）**：

| ステータス | 基礎初期値 | 時間特性 | 主な意味（共通ベースラインD） |
|---|---:|---|---|
| 健康 | 60 | **中長期**（累計の影響大、変化はゆるやか） | 体調・健康寿命に影響。低いと寿命短縮・病気イベント。 |
| 活力 | 50 | **短期**（直近歩数に敏感、歩かないと減衰） | 日々の歩数の短期的勢い。行動力に影響。 |
| 人望 | 50 | 中期（累積寄りだが緩やか） | 人間関係・結婚・人脈に影響。 |
| 財産 | 40 | 中期〜長期（イベント／事業で増減、相続あり） | 就職・事業で増減、次代への相続に影響。 |

実際の初期値 = 基礎初期値 + 遺伝補正（§7-3）+ 家宝補正（§7-4）。算出後は0〜100にクランプ。

### 5-2. 日次更新式（基準＝目標相対）

毎日の日次バッチで、その日の歩数 `steps_today` を各ステータスへ反映する。中立点は固定基準ではなく、**その週の週間目標（activeGoal）を日割りした g** とする（目標相対）。週間目標は週次で変わりうるため、**常にその週のactiveGoalを用いて g を再計算する**。正規化した日次達成度 `d` を用いる。

```text
# g：その週の週間目標(activeGoal)の日割り。週ごとに変わりうるため当週の値を用いる。
g = activeGoal / 7                          # 例：週56000歩→g=8000、週42000歩→g=6000
d = (steps_today - g) / g                   # 例：g比1.5倍→+0.5、半分→-0.5、0歩→-1.0
d = clamp(d, -1.0, +2.0)                    # 過剰歩数の上振れは+2.0で頭打ち

# --- 活力（短期・敏感）---
# 当日の達成度に強く反応。さらに「歩かないと減衰」する自然減衰を持つ。
vitality_decay   = 3                         # 1日の自然減衰（歩いても歩かなくても基礎減衰）
vitality_gain    = round(d * 10)             # d=+1で+10、d=-1で-10
活力 += vitality_gain - vitality_decay
# → 目標どおり(g)歩いても実質 -3（毎日歩き続けないと維持できない短期指標）
#   g×2(d=1.0)なら +10-3 = +7、0歩(d=-1.0)なら -10-3 = -13

# --- 健康（中長期・緩やか）---
# 当日寄与は小さく、累積でじわじわ動く。マイナスにも緩やか。
health_gain = round(d * 2)                    # d=+1で+2、d=-1で-2
健康 += health_gain
# → 1日では大きく動かないが、よく歩く日が続くと着実に上がる

# --- 人望（中期）---
# 歩く＝外に出る＝人と関わる、という間接効果。健康より速く活力より遅い。
charm_gain = round(d * 4)
人望 += charm_gain

# --- 財産（中期〜長期）---
# 日次の歩数寄与は小さい。財産は主に人生イベント(03)で大きく動く。
# 日次は「勤勉ボーナス」程度のごく小さな寄与に留める。
wealth_gain = round(max(d, 0) * 2)            # 基準超過分のみ加点（マイナスにはしない）
財産 += wealth_gain

# --- 共通：クランプ ---
各ステータス = clamp(各ステータス, 0, 100)
```

時間特性の意図：
- **活力**＝短期指標。毎日 `-3` の自然減衰があるため、歩き続けないと下がる。直近の運動の勢いを表す。
- **健康**＝中長期指標。日次寄与は小さい（±2程度）が、累積で大きく動く。寿命（§4）に直結。
- **人望**＝中期。健康と活力の中間の応答速度。
- **財産**＝日次寄与はごく小さく、主に03の人生イベントで増減する（本書は接続点のみ規定）。

### 5-3. 上限・下限と低ステータス時のペナルティ

- すべて0〜100にクランプ。100で頭打ち、0で底打ち。
- **健康が低い場合のペナルティ**（確定しきい値）：
  - 健康 ≤ 30：**病気イベントの誘発フラグ**を立てる（イベント本体・確率は03で定義。本書はフラグ条件のみ規定）。
  - 健康 ≤ 20：老年期突入時（§4-2）の最終寿命計算で `health_bonus` が負に大きく効き、寿命が下限14日へ寄る。
  - 健康 = 0：その状態が続いても**即死はしない**（大往生は寿命到来でのみ発生）。ただし病気イベント誘発フラグは常時オン。救済モードはない（健康低下による短命化は仕様どおりの結果であり、滅亡＝01とは別）。
- **活力が低い場合**：活力 ≤ 20 で行動力低下を表すUI演出（しょんぼり表情等＝§6）。ゲーム的ペナルティは設けず、見た目フィードバックに留める。
- 人望・財産の低下は本書ではペナルティを課さず、03のイベント選択肢の広狭に反映する（接続点のみ）。

### 5-4. 歩数蓄積量（累計歩数）の定義と整理

- **歩数蓄積量（累計歩数 S）** = 当主が誕生してから現在までに歩いた歩数の総和。日次バッチで `S += steps_today` と単調増加する。
- 歩数蓄積量は**ステータスとは別概念**：
  - ステータス（0〜100）は「現在の状態」を表し、増減・減衰・クランプがある。
  - 歩数蓄積量（歩・無上限）は「当代の歩いた総量」を表し、減らない。
- 用途の違い：
  - 歩数蓄積量 → §4の寿命計算、§6の見た目変化、§7の家宝ランク判定（生涯通算達成率＝累計実歩数÷期待累計歩数）、03のイベント結果に使用。
  - ステータス → 日々の状態・寿命の健康補正・03のイベント選択肢に使用。
- **代替わり時、歩数蓄積量はリセット**（次代は0からカウント）。ステータスは遺伝で一部のみ継承（§7）。

## 6. 見た目の変化

### 6-1. 方針
- **ひと目で運動成果が分かる**ことを最優先（企画5.5）。歩数蓄積量とステータスに応じ、体型・姿勢・表情・服装を段階的に変える。
- 変化は「歩いた分だけ好転」の正の表現に統一（やつれ等のネガ表現は活力低下時の軽い演出に留める）。

### 6-2. 体型・姿勢の段階（歩数蓄積量＋健康ベース、5段階）

`fitness_level` を歩数蓄積量 S と健康 H から決める（確定）。

```text
fit_score = clamp(H, 0, 100) * 0.6 + clamp(S / 5000.0, 0, 100) * 0.4
# S=500,000歩で歩数項が満点(100)に到達するスケール
```

| 段階 | fit_score | 体型・姿勢 |
|---:|---|---|
| 1 | 0–20 | 猫背・運動不足体型。だるそうな姿勢。 |
| 2 | 21–40 | 標準やや下。姿勢が少し改善。 |
| 3 | 41–60 | 標準。背筋が伸びる。 |
| 4 | 61–80 | 引き締まった体型。きびきびした姿勢。 |
| 5 | 81–100 | 健脚・矍鑠（かくしゃく）。歩く所作が美しい。 |

### 6-3. 表情の段階（活力ベース、3段階）

短期指標の活力で日々変わり、「最近歩いているか」が一目で分かる。

| 段階 | 活力 | 表情 |
|---:|---|---|
| しょんぼり | 0–20 | 元気がない・伏し目。 |
| ふつう | 21–60 | 穏やかな平常表情。 |
| いきいき | 61–100 | 明るい・はつらつ。 |

### 6-4. 服装の段階（財産ベース、3段階）

財産はイベントで大きく動くため、暮らしぶりを服装で表現。

| 段階 | 財産 | 服装 |
|---:|---|---|
| 質素 | 0–33 | 簡素な普段着。 |
| 標準 | 34–66 | 一般的な装い。 |
| 立派 | 67–100 | きちんとした上質な装い。 |

### 6-5. ステージとの重ね合わせ
- 上記の体型/表情/服装の段階は、**人生ステージ別のベースアート**（幼少期＝子ども体型、老年期＝高齢者体型など）に重ねて適用する。アセットの具体構成は04（画面/アセット）と整合させる。
- MVPでは表情3段階のみ実装でも可（成果フィードバックの最小線）。体型5段階・服装3段階は肉付け段階で追加（企画10）。

## 7. 代替わり処理

### 7-1. 発生条件
- 当主が最終寿命（§4）に達し**大往生**したとき、次代当主を生成して代替わりする。
- 滅亡（01）の場合は代替わりではなく**新一族での再出発**となる（家名も新規・歴史書へ記録）。本節は大往生による継続のみを扱う。

### 7-2. 引き継ぐもの / リセットされるもの

| 区分 | 項目 | 扱い |
|---|---|---|
| **引き継ぐ** | 家名 | そのまま継続（プレイヤー命名／B-2）。 |
| 引き継ぐ | 代数 | **+1**（例：3代目→4代目）。家系図に前当主を1人刻む。 |
| 引き継ぐ | 遺伝（外見・性格の一部） | 各形質を確率継承（§7-3）。 |
| 引き継ぐ | 家宝 | 前当主の生涯通算達成率に応じたランクで次代へ恩恵（金/銀/銅＝+10/+6/+3、必ず銅以上／§7-4）。 |
| 引き継ぐ | ステータスの一部 | 親の最終ステータスの**20%**を次代初期値へ加味（§7-3）。 |
| **リセット** | 歩数蓄積量（累計歩数 S） | **0にリセット**（次代は0からカウント）。 |
| リセット | 活力 | 基礎初期値50から開始（短期指標は持ち越さない）。 |
| リセット | 当代の人生ステージ・経過日数 | 誕生（0日目）から再スタート。 |
| リセット | 病気イベント誘発フラグ等の当代状態 | クリア。 |

### 7-3. 遺伝の計算（確定）

```text
# --- 外見・性格の形質（離散）---
# 形質例：髪型/輪郭/体格傾向/性格タグ など各カテゴリ
for trait in 遺伝対象形質:
    if random() < 0.50:            # 形質継承率50%（§1-5）
        次代.trait = 親.trait      # 親から継承
    else:
        次代.trait = ランダム生成   # 突然変異枠

# --- ステータスの継承（連続・継承率20%）---
INHERIT_RATE = 0.20
for stat in [健康, 人望, 財産]:     # 活力は短期指標のため継承対象外
    親残余 = max(親.最終[stat] - 基礎初期値[stat], 0)   # 基礎超過分のみ継承
    次代.初期[stat] = 基礎初期値[stat] + round(親残余 * INHERIT_RATE)

健康/人望/財産 = clamp(次代.初期[stat], 0, 100)
活力 = 50   # 基礎初期値（リセット）
```

- 「基礎超過分のみ」を継承対象にすることで、よく育てた親ほど次代の出発点が高くなる（負の継承＝親の不調を子に背負わせる、はしない）。
- 当主名・性別は§1-7（B-2）に従い自動生成（任意で改名可）。家名はプレイヤー命名のまま。

### 7-4. 家宝の計算（達成率ベース）

前当主の**生涯通算達成率**（生涯累計実歩数 ÷ 生涯期待累計歩数）で家宝ランクを判定し、次代の初期ステータスへ1代限りの恩恵を付与する。絶対歩数閾値（旧100k/250k/400k）は廃止し、目標相対で評価する。**低ランクでも必ず銅以上＝必ず何かを遺す（空にしない）**。

- **生涯累計実歩数** = 前当主が誕生〜大往生まで歩いた歩数の総和（＝歩数蓄積量 S の最終値）。
- **生涯期待累計歩数** = 各経過日の `g`（その日が属する週の週間目標÷7）を生涯の全実日数ぶん合算した値。週間目標が一定なら `g × final_lifespan`。

| 家宝ランク | 前当主の生涯通算達成率 | 次代初期ステータスへの恩恵 |
|---|---|---|
| 金 | **≥ 110%** | いずれか1ステータスに **+10** |
| 銀 | **80〜110%** | いずれか1ステータスに **+6** |
| 銅 | **< 80%** | いずれか1ステータスに **+3** |

```text
# 1) 生涯通算達成率を算出
lifetime_rate = 生涯累計実歩数 / 生涯期待累計歩数

# 2) ランク判定（必ず銅以上＝下限+3）
if   lifetime_rate >= 1.10: heirloom_bonus = 10   # 金
elif lifetime_rate >= 0.80: heirloom_bonus = 6    # 銀
else:                        heirloom_bonus = 3    # 銅（必ず何か遺る）

# 3) 恩恵の配分先：前当主が最も伸ばしたステータスに対応する家宝種別を遺す
#   健康が最高 → 健康系家宝（健康に加点）
#   財産が最高 → 財産系家宝（財産に加点） など
target = argmax(前当主.最終ステータス[健康, 人望, 財産])
次代.初期[target] += heirloom_bonus   # +10 / +6 / +3
次代.初期[target] = clamp(次代.初期[target], 0, 100)
```

- 家宝の恩恵は**累積させず1代限り**（インフレ防止／§1-6）。家宝そのものの所持表示・家系図への記録は04の家系/家宝表示に委ねる。
- 達成率の期待累計歩数の式（g＝週間目標÷7 × 経過実日数）は本書（02）が正。03の達成率判定（高≥110/中80-110/低<80）と同一の目標相対基準を用いる。

### 7-5. 次代当主の生成手順（まとめ）

```text
1. 前当主を大往生として確定し、家系図に記録（一生のダイジェスト＝04）
2. 代数 += 1
3. 外見・性格形質を確率継承（§7-3）／当主名・性別を自動生成（B-2）
4. ステータス初期値を算出：
     初期[stat] = 基礎初期値[stat] + 遺伝補正(§7-3) + 家宝補正(§7-4)   ※0〜100にクランプ
     活力 = 50（リセット）
5. 歩数蓄積量 S = 0（リセット）
6. 人生ステージ＝誕生(0日目)から開始、当代状態フラグをクリア
7. ホームへ「○○家 N代目」を表示（04）
```
