## 1. 確定した方向性

- **レトロドット調**。顔の情報量を意図的に絞り（目＋口もとのみ）、プレイヤーの想像で補完させる。個性・状態の表現は**顔以外のパーツ（髪・服・小物）のシルエット**が担う。
- **UIテーマ：「和紙と墨」基調＋「宵」演出**
  - 通常画面（ホーム・家系図・歴史書・設定）＝和紙と墨（明色・絵巻物トーン）
  - 演出場面（滅亡・大往生・夜間表現）＝宵の家並み（深藍・提灯橙）
  - 全画面ドットフォント **DotGothic16**（Google Fonts・無料）を使用
- **キャラクター：24×32ドット 2頭身のパーツ合成**（幼少期は24×24）

## 3. キャラクター仕様

### 3-1. 規格

| 項目 | 値 |
|---|---|
| キャンバス | 24×32px（頭16行＋体16行＝2頭身）。幼少期は24×24px（体8行） |
| 表示倍率 | ホーム×4〜6、家系図リスト×2〜3（整数倍・`image-rendering: pixelated`） |
| 実装方式 | 文字グリッド（半身12文字を定義し左右ミラー）＋色キー置換＋座標パッチ。画像ファイル不使用・全パーツがデータ |
| 共通色キー | K輪郭 / S肌 / s肌影 / E目 / H髪 / C服 / c服影 / B帯 / W襟・白 / F袴・靴 / G金 / R赤飾り |

試作実装：`web/dev/sprites.html`（このコードがパーツ定義の現行プロトタイプ）。

### 3-2. パーツ一覧（v1.0時点）

| カテゴリ | 種類 | 内容 |
|---|---|---|
| 髪型 | **9種** | 短髪／つんつん／おかっぱ／長髪／ポニーテール／ツインテール／お団子／まげ／まるぼうず |
| 髪色 | **5色** | 黒・茶・栗・灰・白（白は老年期で自動適用） |
| 服装（形状） | **6形状** | 野良着／着物（無地）／着物（縞）／洋装／羽織／紋付 |
| 服装（色） | 3段階 | 財産段階（02 §6-4）：質素・標準・立派のパレットを形状に乗算 |
| 表情 | 3段階 | 活力（02 §6-3）：ふつう／いきいき（口角＋頬の血色）／しょんぼり（伏し目＋への字） |
| 小物 | 5種＋ | かんざし／リボン／はちまき／杖（老年）／白ひげ（老年） |
| ステージ差分 | 3種 | 幼少（24×24）／成人／老年（白髪＋ひげ＋杖） |

組み合わせ：髪9×髪色4×服6形状×財産3×表情3 ≒ **1,900通り**（＋ステージ・小物）。

### 3-3. ゲーム仕様との対応（02 §6 が正本）

| 見た目 | 駆動する値 | 表現 |
|---|---|---|
| 表情3段階 | 活力（0-20/21-60/61-100） | 目・口もと・頬のパッチ差し替え |
| 服装3段階 | 財産（0-33/34-66/67-100） | 服パレット（質素/標準/立派）。形状は世代の個性として固定 |
| 体型5段階 | fit_score（健康×0.6＋歩数×0.4） | v1.0では**姿勢・体幅の差分は未実装**。段階1-2は猫背パッチ、4-5は肩幅+1px等を肉付けで追加（02 §6-5のMVP最小線＝表情3段階は満たす） |
| 遺伝（外見形質） | 02 §7-3 の50%継承 | **髪型・髪色を形質として継承** → 家系図で血筋が見える |
| 小物 | イベント・家宝由来 | かんざし＝E07良縁／はちまき＝E10挑戦／リボン＝幼少・遺伝など（03のイベント結果と対応付けて肉付け） |

## 4. 実装方針

1. キャラ描画モジュール `web/js/core/avatar.js`（仮）として試作の文字グリッド方式を本実装に移植。コアと同様に**純データ＋純関数**（DOM非依存、canvasは引数注入）とし、Android版（Compose Canvas）に同じグリッドデータを移植できる形にする。
2. 当主データに外見パーツ（髪型・髪色・服形状・小物）を持たせ、遺伝（02 §7-3）の形質対象を「髪型・髪色＋性格タグ」に対応付ける（05 §9-D6系の具体化。性格タグは従来どおりテキスト）。
3. UIテーマはCSSカスタムプロパティを全面置換（和紙と墨）＋演出クラス（宵）で実装。フォントはDotGothic16をセルフホストまたはGoogle Fonts読み込み。
